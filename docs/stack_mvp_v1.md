# TECH01 – Stack del MVP

## Propósito

Registrar las decisiones tecnológicas adoptadas para el MVP y establecer una referencia estable para el desarrollo inicial.

## Decisiones principales

### Framework

- **Elegido:** Next.js + React
- **Motivación:** velocidad de desarrollo, ecosistema maduro, routing integrado y despliegue sencillo en Vercel.

### Backend

- **Decisión:** No habrá backend dedicado en esta fase.
- Se usará únicamente Next.js sin API Routes ni servicios externos.
- No se empleará base de datos.

### Datos del MVP

- **Fuente:** JSON local estático incluido en el repositorio.
- **Contenido:** información fija de la jornada y los 8 jugadores definidos para el experimento.
- **Razón:** simplicidad, velocidad y ausencia de dependencias externas.

## Alcance

Este documento únicamente recoge la elección del stack para el MVP.  
La arquitectura, estructura de carpetas u otros detalles técnicos se documentarán en archivos independientes.

## Estado

Decisiones confirmadas y válidas para iniciar el desarrollo del MVP.
